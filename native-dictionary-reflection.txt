В отличие от эталонного решения, я не добавлял метод size(). Если брать обычный массив, где (size() - 1) это по сути
индекс последнего элемента, и сам size можно использовать в других методах, например узнать есть ли в массиве место
или использовать его для итерирования по массиву, то в словаре, по моему мнению, size не нужен. Больше бы подошла
операция проверки на пустоту isEmpty.
В эталонном примере, команда put имеет два варианта выполнения (постусловия). Это логично, можно добавить новый элемент,
а можно изменить старый. Я, честно говоря, это тоже подразумевал, но не хватило сообразительности чтобы это правильно
написать Т_Т. А раз команда имеет два пути завершения, то нужно было добавить запрос статуса для команды put, я этого
не сделал. До этой задачи, думал что запрос статусов нужно делать только для запросов или команд, для которых есть
предусловия.
